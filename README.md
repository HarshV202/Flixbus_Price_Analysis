# Flixbus Price Analysis — AI Pricing Agent

> **From raw listings to actionable pricing brief — fully automated.**

A two-phase system that identifies pricing anomalies across 850,000+ bus listings and generates a plain-English daily brief using an AI agent — replacing manual analyst review entirely.
Dataset : https://drive.google.com/drive/folders/1QQboE8IV9OP2h0Z3oqwHWC9T3ogG7dQO?usp=sharing

---

## What This Does

Most pricing systems end at a dashboard. This one doesn't.

**Phase 1** runs a similarity-matching and flagging pipeline across 850,000+ Flixbus and competitor listings. It finds every listing where Flixbus is priced too high or too low vs. comparable buses — accounting for load factor, search rank, and peer pool confidence.

**Phase 2** feeds the worst flags into an AI agent (Gemini 2.5 Flash) that writes a structured daily pricing brief: what's wrong, why it matters, and exactly what price action to take.

No dashboard to open. No analyst needed. Raw data → written decision in ~35 seconds.

## Phase 1 — Similarity Matching & Flagging

### Three-Tier Similarity Framework

A competitor bus is comparable to a Flixbus listing only if it passes all three tiers in sequence.

**Tier 1 — Hard Filters (exact match)**
- Route Number
- Departure Date
- Product Key (AC status + bus type combined)

**Tier 2 — Soft Filters (range match)**
- Departure Time: ±90 minutes
- Journey Duration: ±45 minutes

**Tier 3 — Quality Band**
- Rating: ±0.5 stars
- Minimum 50 reviews (excludes unreliable operators)

### Flagging Logic

Base threshold: ±15% deviation from peer median WAP.

A flag is raised only when **both** conditions are breached simultaneously:
- Deviation > 15% from peer median WAP
- Absolute difference > ₹75

**Dynamic adjustments:**

| Signal | Condition | Effect |
|--------|-----------|--------|
| Load Factor | > 80% | Upper threshold widens by +10% |
| Load Factor | < 30% | Lower threshold widens by +10% |
| SRP Rank | Top 20% | Upper threshold widens by +5% |

Adjustments stack — a high-load, top-ranked bus gets a 30% upper threshold before being flagged as TOO HIGH.

**Severity levels** (based on excess beyond threshold):

| Severity | Excess | Action |
|----------|--------|--------|
| CRITICAL | > 30% | Immediate review |
| HIGH | 15–30% | Pricing action recommended |
| MEDIUM | 5–15% | Review and consider |
| LOW | 0–5% | Monitor |

## Phase 2 — AI Pricing Agent

### What It Replaces

Previously: analyst opens Excel → reads hundreds of flagged rows → manually writes pricing brief for the team.

Now: `python pricing_agent.py` → brief is written and saved automatically.

### How It Works

1. **Read** — loads `pricing_analysis_final.xlsx`, filters CRITICAL flags
2. **Rank** — sorts by absolute WAP deviation %, takes top 20 worst flags
3. **Generate** — sends structured flag data to Gemini 2.5 Flash with a prompt that requests per-flag diagnosis and specific ₹ recommendations
4. **Save** — writes `daily_pricing_brief.txt` with date-stamped header

### Sample Output

```

Generated by AI Pricing Agent
============================================================

Executive Summary:
20 CRITICAL flags identified in today's run. Flags are concentrated on
Routes 29, 47, and 48 with deviations of 280–403% above peer median WAP.
Notable: several high-deviation flags on Route 48 show peer pools with
near-identical min/max WAP (single competitor), indicating potential
data quality issues in the comparable pool — budget operators may be
slipping through the similarity filter on premium AC routes.
Recommend validating comparable pool composition on Route 48 before
taking price action.

Route 29 | 25 Mar 2026 | 09:55 | AC Seater
Flixbus WAP ₹2,136 vs peer median ₹518 (312% above). Load factor 7%,
HIGH confidence pool of 7 comparable buses.
→ Investigate comparable pool composition. If peers are genuinely
  comparable, reduce WAP toward ₹900–1,100 range.

Route 47 | 25 Mar 2026 | 21:45 | AC Seater
Flixbus WAP ₹1,975 vs peer median ₹516 (283% above). Top 20% SRP rank,
0% load factor, HIGH confidence pool of 12 comparable buses.
→ High-visibility route with zero bookings. Review peer composition;
  if valid, significant price reduction required to capture demand.
```

The agent also flags data quality issues automatically — in the above run, it correctly identified that Route 48 CRITICAL flags had suspiciously thin comparable pools (1 competitor, identical min/max WAP), which a human reviewer scanning 31,000 rows would likely miss.

## Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| WAP over min/max price | Weighted average reflects actual revenue per bus across fare tiers |
| Dual condition for flags | Prevents false positives on cheap buses (large % = small ₹) and low-priced buses (large ₹ = small %) |
| Median over mean for peer benchmark | Resistant to outlier distortion from premium operators |
| Dynamic thresholds | High-demand, high-visibility buses legitimately command premiums — flat thresholds would over-flag them |
| Top 20 flags for agent | Focuses the brief on what's actionable; avoids token limits and noise |
| Mode for price aggregation | More representative than mean when stores show slightly different prices for same SKU |
